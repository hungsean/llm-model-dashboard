# Issue #5：模型比較

## 背景／為什麼

光看列表還不夠，使用者常想把幾個候選模型抓出來並排比較價格與規格，幫助選型。這張在列表之上加一層「挑幾個來比」的能力。

## 範圍

**要做：**
- 在價格列表（#4）上提供「選取」機制，可勾選 2~N 個模型加入比較。
- 提供比較視圖（並排卡片或對照表），呈現所選模型的 input/output 價格與 context window 等規格。
- 在比較視圖中標出每個比較維度的相對優劣（例如最便宜的 input 價高亮）。
- 可移除已選模型、清空比較。

**不做：**
- 不做成本試算器（依實際用量算錢，另開 Issue）。
- 不做歷史價格走勢。
- 不把比較結果存進後端或產生分享連結。

## 驗收條件

- [ ] 能從列表勾選至少 2 個模型進入比較視圖。
- [ ] 比較視圖並排顯示所選模型的 input 價、output 價、context window。
- [ ] 每個維度的最佳值（如最低 input 價）有視覺標示。
- [ ] 能單獨移除某個模型，也能一鍵清空比較。
- [ ] 手機寬度下比較視圖可正常閱讀。

## 預估大小

**中**（偏小）。

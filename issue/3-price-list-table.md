# Issue #3：價格列表／表格（MVP 核心）

## 背景／為什麼

這是 dashboard 的核心價值：讓使用者一眼看到各家模型的價格。完成這張，產品就有了最小可用的樣子（MVP）。

## 範圍

**要做：**
- 依 #1 決定的 schema 與存放方式，把價格資料載入前端。
- 以表格呈現各模型：至少含 `provider`、模型名稱、input 價格、output 價格、context window。
- 支援**排序**（至少能依 input 價、output 價排序）。
- 支援**篩選**：依 provider 篩選，並有一個關鍵字搜尋框（模型名稱）。
- 單位顯示清楚（例如「USD / 1M tokens」）並標註資料更新時間。
- 基本 RWD，手機上表格可正常閱讀。

**不做：**
- 不做模型並排比較（那是 #4）。
- 不做成本試算器、不做歷史走勢圖。
- 不做使用者帳號 / 收藏 / 後端可寫入的 API。

## 驗收條件

- [ ] 頁面能列出 #1 來源的模型，每列含 provider、模型名、input 價、output 價、context window。
- [ ] 可依 input 價與 output 價排序，點擊欄位即時生效。
- [ ] 可依 provider 篩選，且關鍵字搜尋能即時過濾模型名稱。
- [ ] 價格單位明確標示，頁面顯示資料更新時間。
- [ ] 手機寬度下表格不破版、可閱讀。

## 預估大小

**中**。

## 相依關係

- **依賴 #1**（資料 schema 與存放方式）與 **#2**（專案骨架）。兩者完成後才開工。
